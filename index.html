Introducción

El Problema de la Factorización de Enteros es el pilar sobre el cual se construye gran parte de la seguridad en internet. Su importancia radica en una asimetría fundamental: es una operación que es fácil de hacer, pero increíblemente difícil de deshacer.

A continuación se presenta una introducción estructurada para entender por qué este problema matemático mantiene a salvo tus datos.

// 02

La Intuición: La Función Trampa

En matemáticas, una función de trampa (o one-way function) es aquella que es sencilla de calcular en una dirección, pero casi imposible de invertir a menos que tengas una información especial — la "llave".

→ DIRECCIÓN FÁCIL // MULTIPLICACIÓN

Si te pido multiplicar dos números primos grandes, cualquier computadora lo hace en microsegundos.

61 × 53 = 3,233
← DIRECCIÓN DIFÍCIL // FACTORIZACIÓN

Si solo te doy el número 3,233 y te pido encontrar qué dos primos lo originaron, tendrías que probar divisiones una por una. Con 600 dígitos, las computadoras más potentes tardarían siglos.

// 2.3  Código en Octave

OCTAVE // Primos.m 
% Script interactivo de Factorización RSA en Octave
pkg load communications; % Asegura el acceso a funciones matemáticas

printf("--- GENERADOR DE CLAVE RSA (SIMULADO) ---\n");

% 1. Entrada de datos por el usuario
p = input("Introduce el primer número primo (p): ");
q = input("Introduce el segundo número primo (q): ");

% 2. Validación: ¿Son ambos números primos?
if isprime(p) && isprime(q)

  % Operación de cifrado (Fácil)
  N = p * q;
  printf("\n[PROCESO]: Ambos son primos. Generando clave pública...\n");
  printf("Producto N (Clave Pública): %d\n", N);

  % 3. Intento de Factorización (Difícil)
  printf("\n--- INICIANDO ATAQUE DE FUERZA BRUTA ---\n");
  printf("Buscando los factores de %d sin conocer p ni q...\n", N);

  tic; % Iniciar cronómetro
  factor_encontrado = [];

  % Algoritmo de división por tentativa hasta la raíz cuadrada de N
  limite = floor(sqrt(N));
  for i = 2:limite
    if rem(N, i) == 0
      factor_encontrado = [i, N/i];
      break;
    end
  end
  tiempo = toc;

  % 4. Resultados
  if ~isempty(factor_encontrado)
    printf("\n¡ÉXITO! Se ha roto el cifrado.\n");
    printf("Factores descubiertos: %d y %d\n", factor_encontrado(1), factor_encontrado(2));
    printf("Tiempo total de cómputo: %.6f segundos\n", tiempo);
  else
    printf("\nERROR: No se pudieron encontrar los factores.\n");
  end

else
  % Respuesta si el usuario no ingresó primos
  printf("\n[ERROR]: Uno o ambos números no son primos.\n");
  if ~isprime(p); printf("- %d no es un número primo.\n", p); end
  if ~isprime(q); printf("- %d no es un número primo.\n", q); end
  printf("Por favor, intenta de nuevo con números primos.\n");
end

¿Cómo lo usa el algoritmo RSA?

El sistema RSA (llamado así por sus inventores Rivest, Shamir y Adleman) utiliza esta dificultad para crear un sistema de llave pública y llave privada.

🔓
LLAVE PÚBLICA
Es el resultado de p × q = N. Este número se entrega a todo el mundo. Cualquiera puede usarlo para cifrar un mensaje.
🔐
LLAVE PRIVADA
Son los números originales p y q. Solo tú los conoces. Necesarios para descifrar.
⚙️
EL SECRETO
Para descifrar se necesita la función de Euler, calculable solo con p y q separados.
φ(N) = (p − 1)(q − 1)
Solo calculable eficientemente cuando se conocen p y q por separado. Sin ellos, el descifrado es imposible.
// 2.6

¿Por qué es un Problema Abierto?

◈
SIN PRUEBA DE IMPOSIBILIDAD

Nadie ha demostrado que no exista un algoritmo rápido para factorizar en computadoras clásicas. Simplemente, después de décadas de investigación, nadie lo ha encontrado.

◈
COMPLEJIDAD COMPUTACIONAL

En la teoría de la complejidad, la factorización no se ha clasificado definitivamente. Se sabe que no es "tan fácil" (clase P), pero tampoco se ha demostrado que sea de los más difíciles (clase NP-completo).

// 2.7

La Amenaza en el Horizonte: El Factor Cuántico

⚠ ALERTA CUÁNTICA // AMENAZA CRÍTICA

El mayor desafío para este problema es el Algoritmo de Shor. Este algoritmo matemático demuestra que una computadora cuántica lo suficientemente grande podría factorizar números gigantes en minutos.

Si se construye una computadora cuántica capaz de manejar miles de qubits estables, el "Problema de la Factorización" dejaría de ser un problema, y el sistema RSA quedaría obsoleto instantáneamente. Por eso, hoy se trabaja en la Criptografía Post-Cuántica, que busca problemas matemáticos aún más difíciles que la factorización.
